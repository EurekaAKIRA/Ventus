# 项目介绍文档（论文版）

## 一、项目名称
面向 API 自动化测试的智能需求驱动平台设计与实现

---

## 二、项目背景与研究意义
随着软件系统迭代速度持续提升，传统依赖人工编写测试用例的 API 测试方式逐渐暴露出效率低、覆盖不稳定、维护成本高等问题。尤其在需求频繁变更、接口链路复杂、跨团队协作紧密的场景下，测试设计与执行之间存在明显断层：需求文档难以直接转化为可执行测试资产，导致“文档—用例—执行—报告”链路割裂。

本项目围绕这一痛点，构建了一个以需求文档为输入、能够自动完成场景生成、DSL 构建、执行验证与结果分析的智能化测试平台。项目意义主要体现在：

1. 提升测试资产生产效率，降低手工编写和维护成本；
2. 强化需求到测试的可追溯性，减少理解偏差；
3. 建立可复用、可扩展的自动化测试工程体系；
4. 为智能测试平台化落地提供可实践的架构与方法论。

---

## 三、项目目标
本项目的总体目标是实现“需求驱动测试生成与执行闭环”，具体包括：

- 将 API 需求文档自动解析为结构化需求信息；
- 自动生成测试场景、步骤与断言逻辑；
- 形成统一 DSL 以支持稳定执行；
- 实现任务执行、结果回传、分析报告与归档管理；
- 支持规则与大模型协同，兼顾可控性与智能性。

---

## 四、总体架构设计
系统采用工作流编排为主、结合自定义 Agent 能力增强的混合架构。整体由任务中心统一调度，按照“需求输入→解析→生成→执行→分析→归档”的流程串联各模块，关键节点引入自定义 Agent 智能能力以增强复杂语义理解与断言规划效果。

### 4.1 架构分层
- 接入层：任务创建、参数配置、结果查询等 API/页面入口。
- 编排层：任务中心负责流程控制、状态流转与工件管理。
- 能力层：需求分析、用例生成、执行引擎、结果分析等核心能力模块。
- 存储层：任务数据、解析产物、执行记录、分析报告与归档工件。
- 智能增强层：模型网关、规则引擎、检索增强（RAG）等。

### 4.2 核心模块（中文命名）
1. 任务中心模块：统一任务入口、生命周期管理、流程编排与工件落盘。
2. 需求分析模块：需求文本清洗、结构化解析、接口要素抽取、元信息生成。
3. 用例生成模块：场景构建、步骤展开、断言规划、DSL 生成。
4. 执行引擎模块：HTTP 执行、上下文变量传递、断言校验、失败收敛与重试。
5. 结果分析模块：执行结果聚合、失败分类、质量评估与报告生成。
6. 共享能力模块：模型网关、配置管理、统一策略与通用工具能力。

---

## 五、关键流程设计（端到端闭环）

### 5.1 任务创建
用户提交任务名称、需求文本/文档、目标系统与环境参数，平台生成任务上下文并分配唯一任务标识。

### 5.2 需求解析
系统对原始需求进行清洗、分段与要素抽取，输出结构化需求对象（接口清单、依赖关系、预期效果、约束条件等）。

### 5.3 场景与 DSL 生成
根据解析结果构建测试场景，自动补齐步骤级请求信息、断言条件和上下文保存规则，产出可执行 DSL。

### 5.4 执行调度
执行引擎按 DSL 逐步发送请求，处理 Header 自动补全、统一 envelope 解析、上下文变量注入与断言验证，并实时记录执行结果。

### 5.5 结果分析与报告
系统汇总场景通过率、步骤失败原因、断言命中情况等，生成结构化分析报告，为问题定位与需求改进提供依据。

### 5.6 归档与可追溯
任务相关工件（原始输入、解析结果、场景、DSL、执行结果、报告）统一归档，支持复盘、回归与审计。

## 六、核心技术方案

### 6.1 后端技术
- Python 为主要开发语言；
- 模块化服务实现编排与能力解耦；
- 模型网关支持规则解析与大模型增强协同；
- 执行引擎支持接口执行、断言与上下文管理。

### 6.2 前端技术
- Vue3 构建页面能力；
- 支持任务管理、解析查看、DSL 预览、执行跟踪与报告展示；
- 提供测试统计可视化（通过率、失败分类、场景分布等）。

### 6.3 数据与存储
- MySQL 存储任务元数据与业务记录；
- 文件化工件存储解析/生成/执行产物；
- Redis 用于缓存与性能优化（按部署策略启用）。

---

## 七、项目特色与创新点
1. 需求直达执行：将文档输入与测试执行打通，形成自动化闭环。
2. 规则+模型融合：在可控规则基础上引入智能增强，平衡稳定性与泛化能力。
3. 统一 DSL 抽象：解耦“生成逻辑”和“执行逻辑”，便于扩展与复用。
4. 工件全链路沉淀：支持全流程可追溯、可复盘、可回归。
5. 工程化可落地：具备模块边界清晰、测试覆盖持续、迭代成本可控等工程优势。

---

## 八、测试验证与阶段性成果
项目已围绕主链路开展多轮验证，覆盖任务创建、需求解析、DSL 生成、执行、报告与归档等流程。阶段性成果包括：

- 主链路可稳定跑通并形成标准工件；
- 关键能力（如自动请求头、envelope 兼容、JSON 解析）具备专项测试保障；
- 能够支持外部标准 API 场景（如 Restful Booker）进行端到端验证；
- 平台具备持续回归与质量评估基础。

---

## 九、应用价值
- 对测试团队：降低脚本编写与维护负担，提高回归效率；
- 对研发团队：将需求变更快速转化为可执行验证，缩短反馈周期；
- 对管理侧：提升测试过程透明度与质量可度量能力，支撑质量治理。

---

## 十、存在问题与改进方向
当前系统仍存在以下可优化空间：

1. 复杂语义需求下的断言泛化能力仍需提升；
2. 多接口跨域依赖场景的自动推理精度可继续增强；
3. 部分异常网络/接口行为的鲁棒策略有待细化；
4. 智能能力效果评估指标体系需进一步标准化。

后续可从以下方向推进：

- 引入更细粒度的策略路由与多模型协同；
- 强化失败样本驱动的自适应优化机制；
- 完善可解释报告与质量评分体系；
- 推进平台化多租户与更高并发支持。

---

## 可直接用于论文的总结段
本项目面向 API 自动化测试场景，构建了一个以工作流编排为主、结合自定义 Agent 能力增强的智能测试平台。系统实现了从需求文档输入到场景生成、DSL 执行、结果分析与归档的端到端闭环，兼顾工程可控性与智能扩展能力。实践表明，该平台能够有效提升测试生成效率与执行一致性，为智能化测试体系建设提供了可落地的技术路径与工程参考。

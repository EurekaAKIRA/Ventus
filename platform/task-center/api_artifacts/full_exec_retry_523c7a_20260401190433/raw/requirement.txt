# Platform 端到端任务流需求文档

## 1. 目标

验证平台后端从任务创建到报告查询的主链路可用性，确保任务状态可追踪、产物可读取、解析元数据完整。

## 2. 接口清单

| 接口 | 方法 | 路径 |
|------|------|------|
| 创建任务 | POST | /api/tasks |
| 解析任务 | POST | /api/tasks/{task_id}/parse |
| 获取任务详情 | GET | /api/tasks/{task_id} |
| 获取结构化需求 | GET | /api/tasks/{task_id}/parsed-requirement |
| 获取场景列表 | GET | /api/tasks/{task_id}/scenarios |
| 获取 DSL | GET | /api/tasks/{task_id}/dsl |
| 获取 Feature | GET | /api/tasks/{task_id}/feature |
| 启动执行 | POST | /api/tasks/{task_id}/execute |
| 获取执行结果 | GET | /api/tasks/{task_id}/execution |
| 获取校验报告 | GET | /api/tasks/{task_id}/validation-report |
| 获取分析报告 | GET | /api/tasks/{task_id}/analysis-report |
| 获取产物列表 | GET | /api/tasks/{task_id}/artifacts |
| 删除任务 | DELETE | /api/tasks/{task_id} |

## 3. 主流程

1. 调用 `POST /api/tasks` 创建任务，请求体包含 `task_name`、`source_type`、`requirement_text`、`environment` 字段。
2. 创建成功后，系统返回 `task_id`，任务状态为 `received`。
3. 调用 `POST /api/tasks/{task_id}/parse` 执行解析，请求体包含 `use_llm`、`rag_enabled`、`retrieval_top_k`、`rerank_enabled` 字段。
4. 解析成功后，调用 `GET /api/tasks/{task_id}` 获取任务详情，并确认 `parse_metadata` 存在。
5. 调用 `GET /api/tasks/{task_id}/parsed-requirement` 获取结构化需求，确认包含 `objective`、`actions`、`expected_results`。
6. 调用 `GET /api/tasks/{task_id}/scenarios` 获取场景，确认至少存在 1 个场景且步骤完整。
7. 调用 `GET /api/tasks/{task_id}/dsl` 获取 DSL，确认包含 `scenarios` 和 `steps`。
8. 调用 `GET /api/tasks/{task_id}/feature` 获取 feature 文本，确认包含 `Feature:` 和 `Scenario:`。
9. 调用 `POST /api/tasks/{task_id}/execute` 启动执行，请求体包含 `execution_mode`、`environment` 字段。
10. 调用 `GET /api/tasks/{task_id}/execution` 查询执行状态。
11. 调用 `GET /api/tasks/{task_id}/validation-report` 与 `GET /api/tasks/{task_id}/analysis-report` 查询报告。
12. 调用 `GET /api/tasks/{task_id}/artifacts` 查询产物列表，确认存在 `parsed-requirement`、`scenarios`、`dsl` 类型。
13. 调用 `DELETE /api/tasks/{task_id}` 清理任务。

## 4. 断言

- 创建任务后必须返回可用 `task_id`。
- 解析后 `parse_metadata.retrieval_mode` 应为 `vector+keyword` 或 `keyword`。
- 结构化需求必须包含动作与预期结果。
- 场景与 DSL 必须可查询且不为空。
- 报告接口必须可查询并返回标准响应包。
- 任务删除后再次查询任务应返回不存在。

## 5. 约束

- 所有接口应返回统一响应结构：`success`、`code`、`message`、`data`、`timestamp`。
- 任务解析过程应记录性能与检索相关元数据。
- 当向量检索不可用时，应回退关键词检索且保留回退原因。
